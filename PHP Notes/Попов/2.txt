1. Витягування файлів з БД
	- Якщо я вказав недобру назву стовбчика таблиці, буде помилка і всі подальші операції зі змінною, якій присвоєна ф-ція витягування елементів з таблиці, ббуде помилка.
	- Якщо таблиця пуста, а я прошу від неї елементи, помилки не буде, таблиця поверне пусті елементи.
2. Ф-ція mysql_num_rows(); - к-сть рядків у таблиці БД
3. exit(mysql_error()); - якщо при з’єднанні з БД є якась помилка, ця ф-ція покаже її код.
4. Якщо файл підключений до файла, який підключений до БД, то його вже не треба підключати.
	- І взагалі підключений код нічим не відрізняється, тож треба бути уважним зі ЗМІННИМИ.
5. При передаванні значень форми наприклад методом POST, name - ніби імя знінної, а value її значення.
6. Ф-ція stripslashes(); - удалить(або обробить на безпечні) всі слеші зі строки // stripslashes($name);
7. Ф-ція htmlspecialchars(); - скаже PHP, що це не HTML код, а просто строка. // htmlspecialchars($zm);
8. DISTINCT - унікальні. Якщо я не хочу вибирати одинакові дати з БД - SELECT DISTING date FROM ...
9. Можна вибрати з таблиці з ячейки БД указану к-сть символів, тільки треба вказати з якого боку починати - left(data,7) - з стовбчика data буде брати тільки 7 символів з ліва. Якщо так залишити, буде помилка, треба дати псевдонім скороченим даним через оператор AS - SELECT left(date,7) AS new-zminna from ... І тепер зі скороченими даними працювати через new-zminna і воспринімати це так, ніби у таблиці БД зявився новий стовбчик new-zminna.
10. Попов казав, щоб шукати дані в БД, їх треба проіндексіровати. Просто дати полю свойство індекс. Но я не впевнений, бо текст почав чекатись тільки тоді, коли я дав йому свойство "Полнотекстовый".
11. Щоб шукати дані в БД є  MATCH() AGAINST(). // SELECT * FROM data WHERE MATCH(text) AGAINST('$search')" - буде означати, якщо у стовбчика text таблиці data,  є таке саме як у змінній $search, то вибрати з цієй строки всі дані.
12. Щоб знати к-сть запрашуємих елементів в таблиці, є " COUNT(*) " // $result00 = mysql_query("SELECT COUNT(*) FROM data WHERE cat='$cat'"); - у змінній $result00 буде к-сть елементів у таблиці data, у яких у стовбчику cat значення $cat.
13. Щоб вивести якусь к-сть елементів з таблиці БД, починаючи з конкретного елементи, можна використати 'LIMIT' // $result = mysql_query("SELECT id,title,description,date,author,mini_img,view FROM data ORDER BY id LIMIT $start, $num",$db); - вибере з таблиці data '$num' - к-сть рядків, починаючи з '$start' рядка. Не знаю, чи ORDER BY обов’язкове.
14. Методом POST мош передавати елементи і в той самий файл, не обов’язково спеціально для форми создавати новий файл.
